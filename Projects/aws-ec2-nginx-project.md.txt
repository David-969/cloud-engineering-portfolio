Deployment Workflow

Step 1 — Launch EC2 Instance

Created AWS EC2 instance
Configured Security Groups
Allowed:
SSH Port 22
HTTP Port 80

Step 2 — Install Nginx

Installed Nginx:

sudo dnf install nginx -y

Started Nginx:

sudo systemctl start nginx
sudo systemctl enable nginx
Step 3 — Upload Website Files

Initially deployed files using SCP:

scp -i key.pem -r website-folder ec2-user@PUBLIC-IP:/home/ec2-user

Moved files into Nginx web directory:

sudo cp -r /home/ec2-user/fesh-home/* /usr/share/nginx/html/

Step 4 — GitHub Deployment Workflow

Initialized Git repository locally:

git init
git add .
git commit -m "Initial commit"

Connected to GitHub:

git remote add origin REPOSITORY_URL
git push -u origin main

Cloned repository on EC2:

git clone REPOSITORY_URL

Updated live website:

git pull
sudo cp -r * /usr/share/nginx/html/
sudo systemctl restart nginx
Challenges and Troubleshooting
SSH Timeout Issue

Problem

SSH connection timed out after restarting EC2 instance.

Cause

Security Group SSH rule restricted access to outdated public IP.

Solution

Updated inbound SSH rules to allow current IP access.

Public IP Change

Problem

Website became inaccessible after EC2 stop/start.

Cause

AWS assigned a new public IP address.

Solution

Used updated public IP and learned Elastic IP concepts.

Website Not Loading


Problem

Browser could not access website.

Cause

Used HTTPS instead of HTTP.

Solution

Accessed website using:
http://PUBLIC-IP
